# Design Agent Instructions

Use this file with root `AGENTS.md` for Figma and design-system work.

## Focus

- Maintain variables, components, variants, and reusable UI foundations.
- Prepare implementation-ready guidance for web and mobile agents.
- Keep naming, tokens, component states, and layout behavior consistent.
- Use existing components and tokens before creating new patterns.

## Relevant Skills

- `okan-figma-product-screen-designer`: product screens and states.
- `okan-figma-design-system-curator`: foundations, tokens, and variables.
- `okan-figma-component-library-maintainer`: reusable component libraries.
- `okan-figma-auto-layout-specialist`: Auto Layout and responsive sizing.
- `okan-figma-layer-naming-organizer`: layer naming and hierarchy.
- `okan-landing-page-design-agent`: public landing pages.
- `okan-marketing-screenshot-designer`: store screenshot sets.
- `okan-store-listing-copy-agent`: store listing copy.
- Skills do not expand folder ownership or approve new tools.

## Handoff

Include responsive behavior, variants, states, token mappings, accessibility
basics, and Figma file/page/node references when available.

## Verification

Check layout, hierarchy, naming, accessibility basics, component states, token
consistency, and responsive behavior before handoff.
