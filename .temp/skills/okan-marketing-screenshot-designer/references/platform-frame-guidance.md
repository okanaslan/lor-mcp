# Platform Frame Guidance

Use this when creating platform-specific screenshot frames in Figma.

## Frame Organization

- Separate App Store and Google Play sets.
- Use stable export names: `App Store / Screenshot 01`, `Google Play / Screenshot 01`.
- Keep alternate concepts clearly named: `App Store / Screenshot 01 / Variant B`.
- Preserve approved screenshot sets unless replacement is requested.

## Platform Notes

- Ask for exact target dimensions when the user needs production export sizes and they were not provided.
- If exact dimensions are not specified, create review-ready Figma frames and label them as draft sizing.
- Keep text and important UI away from edges and device notches.
- Use consistent device/mockup treatment inside each platform set.
- Build with export-readiness in mind: clear frame names, no hidden dependencies, no clipped content.

## Mockup Rules

- Reuse real app screenshots or faithful app-screen compositions.
- Do not imply unsupported platforms or unavailable device form factors.
- Keep the visible app state aligned with current product behavior and store copy.
