# Figma Landing Workflow

Use this workflow for Figma-only landing-page work.

## Inspect

- Check existing website, marketing, app, and brand Figma files if available.
- Identify approved frames, visual direction, product screenshots, app preview compositions, and current public copy.
- Confirm whether the user wants alternatives or an in-place update.

## Design

- Preserve approved frames unless explicitly asked to replace them.
- Place alternatives near the source frame with clear names.
- Use a public website composition, not an in-app screen composition.
- Reuse real app screenshots or faithful app-preview cards; avoid inventing product states that conflict with the app.
- Keep store/download/waitlist CTAs visually prominent.

## Layout

- Use desktop-first frames unless the user requests responsive variants.
- Add mobile or tablet variants only when requested or when responsive behavior is the focus.
- Use `Hug contents` for sections, text groups, cards, buttons, and footer link groups.
- Use `Fill container` for full-width sections and columns inside Auto Layout parents.
- Fixed sizes are appropriate for desktop canvases, screenshot masks, device mockups, and export targets.

## Validate

- Check hierarchy from hero to CTA to proof to footer.
- Check clipped text, missing fonts, collapsed frames, and unexpected fixed heights.
- Check footer/legal links exist when required.
- Confirm no code implementation was performed.
