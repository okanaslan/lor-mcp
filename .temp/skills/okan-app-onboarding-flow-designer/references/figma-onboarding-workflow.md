# Figma Onboarding Workflow

Use this workflow for onboarding screens and flow variants in Figma.

## Inspect

- Check the existing Onboarding page, related Auth screens, and first destination product screens.
- Identify the app shell, typography, components, variables, and page-specific visual patterns.
- Confirm whether the user wants alternatives or in-place updates.

## Design

- Preserve approved frames unless the user asks to replace or update them.
- Place alternatives beside the source frame or in the Onboarding page's existing review area.
- Reuse existing components, buttons, inputs, cards, permission copy patterns, and illustration/icon style.
- Keep copy realistic and tied to actual product capabilities.
- Include clear primary and secondary actions for each step.

## Layout

- Use fixed sizing for mobile viewport frames.
- Use `Hug contents` for cards, buttons, inputs, text stacks, and option rows.
- Use `Fill container` for full-width fields, rows, and content columns inside Auto Layout parents.
- Taller-than-viewport review frames are acceptable when documenting full scrollable onboarding flows.

## Validate

- Check step order, CTA consistency, skip/defer visibility, and handoff destination.
- Check clipped text, missing fonts, collapsed frames, broken instances, and unexpected fixed sizing.
- Confirm frame names clearly indicate flow step and state.
