# Onboarding Copy And Handoff

Use this when writing onboarding copy or implementation notes.

## Copy Rules

- Keep headlines short and action-oriented.
- Explain benefits in app-specific language, not generic productivity claims.
- Make setup fields feel useful, not administrative.
- For permissions, explain the user value before the system prompt.
- For AI or automation features, keep the user in control and avoid overpromising.
- Avoid placeholder copy unless the user explicitly asks for wireframes only.

## Handoff Notes

Document only what implementation needs:

- Step order and navigation behavior.
- Required, optional, skipped, and deferred steps.
- Field names, default values, and validation expectations when visible in the design.
- Permission prompt timing and denied/deferred behavior if represented.
- Account, guest, subscription, or locked-state assumptions.
- Completion destination and first-run product state.
- Explicit out-of-scope items.
