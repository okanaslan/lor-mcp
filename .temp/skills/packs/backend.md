# Backend Skill Pack

Use this pack for backend agents working on Go APIs, HTTP contracts, persistence
boundaries, and verification.

## Skills

- `okan-backend-golang-agent`: start or normalize a Go API service.
- `okan-backend-api-domain-agent`: work on handlers, contracts, services,
  repositories, persistence, tests, and OpenAPI-related backend changes.
- `okan-backend-api-slice-implementer`: implement one backend API vertical
  slice end to end.
- `okan-backend-migration-safety-reviewer`: review SQL migration safety.
- `okan-go-integration-test-writer`: add or improve Go integration tests.
- `okan-openapi-contract-maintainer`: keep API contracts and generated OpenAPI
  artifacts synchronized.
- `okan-backend-service-standards`: enforce Monetari/Okan NestJS service and
  feature-module structure.
- `okan-backend-usecase-pattern`: enforce Monetari/Okan NestJS usecase design.
- `okan-backend-env-config`: enforce Monetari/Okan NestJS environment and config
  conventions.
- `okan-test-naming`: classify and name backend integration, e2e, and live tests.
- `find-skills`: discover additional installable backend skills only when
  needed.
- `caveman`: optional terse communication mode only when the user asks for it.

## Notes

- Use the assigned folder's local `AGENTS.md` before invoking a skill.
- Keep backend work inside the assigned backend folder unless the PM expands the
  boundary.
- Do not use this pack as automatic routing logic.
