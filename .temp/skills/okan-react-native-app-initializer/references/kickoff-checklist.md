# Kickoff Checklist

Collect these before scaffolding or changing app identity:
- App display name
- App slug / directory name
- Android package ID
- iOS bundle ID
- Expo account owner, if EAS will be used
- Target platforms: Android, iOS, web
- API base URL strategy: local, staging, production
- Auth requirements for v1
- Push notification requirements for v1
- Deep link / universal link requirements for v1
- Billing or in-app purchase requirements for v1
- Release targets: internal testing, TestFlight, Play Console, App Store

Defaults when the user has no preference:
- Expo + TypeScript
- Expo Router
- Android and iOS enabled
- EAS profiles for development, preview, and production
- No committed credentials or native signing files
