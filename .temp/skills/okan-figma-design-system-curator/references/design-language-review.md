# Design Language Review Guidance

Use this when creating or updating foundation documentation in Figma.

## Documentation Shape

- Keep the guide compact and review-oriented, not a duplicate component library.
- Lead with selected directions, then keep alternatives as decision history.
- Show representative examples, not every component or screen variant.
- Pair swatches with token names and exact values.
- Use short notes that state the rule and when to use it.

## Review Boards

- Name alternatives clearly, such as `Alternative 03 / Butter Shell`.
- Keep each card structurally comparable: swatches, values, preview, and rule summary.
- Mark selected alternatives without deleting other useful decision history.
- Keep exploratory fills local unless a token migration has been approved.

## Handoff Notes

- State whether values are documentation-only, exploratory, or applied to variables.
- Call out affected token names and explicitly list unchanged token families when that prevents confusion.
- Include layout guidance when relevant: use `Hug contents`, `Fill container`, and fixed sizing intentionally.
- Confirm validation checks for clipped text, collapsed frames, missing fonts, and variable binding issues.
