# Component Inventory Checklist

Use this checklist when auditing reusable Figma component coverage.

## Shared Components To Check

- Buttons: primary, secondary, tertiary/link, destructive, icon-only, small action chips.
- Inputs and controls: text fields, notes fields, dropdown rows, steppers, segmented controls, chips, toggles.
- Navigation: tab bar, headers, filter headers, segmented tabs, close/back affordances.
- Cards: task, routine, habit, limit, settings, subscription, inbox brief, report, empty state, upsell.
- Sheets and overlays: bottom-sheet shell, dim overlay, modal cards, handles, sheet CTAs.
- Feedback: in-app notifications, semantic feedback cards, loading skeletons, success/error states.
- Data visuals: habit/limit charts, report metric tiles, status rows, progress/status bars.
- Reusable cells: settings rows, task action cells, remaining-status rows, list rows, metadata rows.

## Audit Questions

- Is this pattern reused across multiple pages or likely to be reused?
- Is there already a canonical component master or component set?
- Are active instances still connected to a master?
- Are there duplicate local masters with the same visual role?
- Are loading, empty, selected, disabled, and error states represented where needed?
- Are component and layer names explicit enough for handoff?
