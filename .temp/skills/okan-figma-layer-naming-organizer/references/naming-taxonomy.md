# Naming Taxonomy

Use stable, semantic names that describe the role of the layer.

## Structural Names

- `Header`
- `Content`
- `Body`
- `Footer`
- `Section`
- `List`
- `Card`
- `Row`
- `Sheet`
- `Modal`
- `Overlay`
- `Preview`

## Role Names

- `Title`
- `Subtitle`
- `Body Copy`
- `Label`
- `Helper Text`
- `Metadata`
- `Value`
- `Badge`
- `CTA`
- `Primary Action`
- `Secondary Action`
- `Trailing Icon`
- `Status Icon`
- `Icon Container`
- `Text Stack`
- `Metadata Row`
- `Action Group`

## Product Area Names

- `Task Card`
- `Routine Card`
- `Habit Row`
- `Limit Row`
- `Settings Row`
- `Inbox Action Cell`
- `Weekly Report Card`
- `Bottom Sheet`
- `Tab Bar`
- `Screen Header`
- `Empty State`
- `Loading State`

## Naming Rules

- Prefer noun phrases over implementation names.
- Include state only when it helps distinguish variants, such as `Loading Row` or `Selected Chip`.
- Include direction only for interaction examples, such as `Left Swipe Action` or `Right Swipe Action`.
- Keep names short enough to scan in the Figma layers panel.
- Match nearby approved naming when a file already has a consistent convention.

## Avoided Names

Avoid these for important layers:

- `Frame`, `Frame 1`, `Frame 12`
- `Group`, `Group 3`
- `Rectangle`, `Rectangle 5`
- `Vector`, except unavoidable raw icon internals
- `Layer`
- `Copy`
- `Wrapper`
- `Container` without a meaningful role prefix
- Autogenerated labels that do not describe UI purpose
