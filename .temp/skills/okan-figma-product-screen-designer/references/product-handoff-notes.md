# Product Handoff Notes

Use this when a Figma screen needs implementation-facing notes.

## Document When Needed

- Screen purpose and user state.
- Primary and secondary actions.
- Navigation entry and exit behavior.
- Data fields displayed or edited.
- Empty/loading/error behavior.
- Gating or authentication assumptions.
- Gesture behavior, if any.
- What is intentionally out of scope.

## Handoff Style

- Keep notes short and close to the relevant frame when useful.
- Use product terms that match the UI and API/model language.
- Avoid speculative implementation details unless they prevent ambiguity.
- If a design uses future or unavailable data, mark it clearly as a requirement or assumption.
