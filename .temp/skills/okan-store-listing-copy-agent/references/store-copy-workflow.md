# Store Copy Workflow

Use this workflow for writing or revising App Store and Google Play listing copy.

## Inspect

- Review product screens, website copy, marketing notes, feature lists, and existing listing copy if available.
- Identify real capabilities, planned capabilities, unavailable features, platform support, pricing/subscription state, and AI behavior.
- Note any risky or unconfirmed claims before drafting.

## Draft

- Start with shared positioning: product category, target user, core promise, and differentiators.
- Write platform-specific variants instead of one generic paragraph for every surface.
- Keep first lines concrete: what the app is, who it helps, and what users can do.
- Use feature bullets only when they add scan value.
- Avoid filler phrases such as “boost your productivity” unless grounded in concrete product behavior.

## Review

- Check character-sensitive fields for practical length even when exact limits are not requested.
- Flag unsupported claims rather than silently removing important product intent.
- Provide a recommended version plus alternatives when useful.
- Keep final copy easy to paste into store consoles.

## Output Shape

A useful copy pack usually includes:

- Shared positioning.
- App Store metadata.
- Google Play metadata.
- Feature bullets.
- What’s new / update notes when requested.
- Claim-risk notes and assumptions.
