# Store Listing Copy Agent

Use this skill for App Store and Google Play copy, metadata, and optional Figma copy boards.

## Operating Model

- Start by inspecting existing product, website, marketing, store-copy, and app-feature context before writing.
- Produce a structured copy pack first; create or update a Figma copy board only when requested or when a marketing Figma file is part of the task.
- Keep claims realistic, platform-safe, and aligned with current or explicitly planned product behavior.
- Prefer concrete user benefits over generic productivity claims.
- Flag unsupported or risky claims involving AI, subscriptions, privacy, health, guarantees, availability, automation, or platform support.

## Scope

Use this skill for:

- App Store app name, subtitle, promotional text, keywords, description, and what’s-new copy.
- Google Play title, short description, full description, update notes, and listing experiment variants.
- Shared positioning, feature bullets, target-user copy, and claim-risk notes.
- Platform-specific copy variants and concise metadata packs.
- Figma review boards that organize store listing copy as editable text.

Do not use this skill for:

- Store screenshot design, app preview visuals, or screenshot headline layout.
- Landing pages, public website design, social posts, ads, or launch graphics.
- In-app product screens, onboarding flows, token systems, component-library cleanup, frontend code, backend, or mobile implementation.

## Copy Rules

- Write for store visitors who do not already understand the product.
- Keep titles and short descriptions clear, concrete, and easy to scan.
- Tie benefits to real product capabilities and avoid unsupported promises.
- Use AI wording only when the product actually uses AI, and keep the user in control.
- Do not imply a web app, platform availability, pricing, privacy status, or compliance status unless confirmed.
- Separate final recommended copy from alternatives and notes.

## Figma Copy Board Rules

- If writing to Figma, follow the required Figma execution skill before any Figma write action.
- Keep Figma boards text-first and editable; do not turn them into screenshot design boards.
- Organize copy by platform: App Store, Google Play, Shared Positioning, Claim Notes.
- Name frames clearly, such as `Store Listing / Copy Pack`, `Google Play / Copy Drafts`, or `App Store / Metadata Variants`.
- Preserve existing approved copy boards unless the user explicitly asks to replace them.
