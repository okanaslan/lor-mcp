# Process Agent Instructions

Use this file with root `AGENTS.md` for commit messages, pull request text,
review comments, status updates, and short operational communication.

## Relevant Skills

- `okan-commit-message`: suggest or create scoped conventional commits.
- `okan-pr-description`: generate concise pull request descriptions.
- `okan-create-review-comments`: write selected inline review comments.
- `okan-respond-process`: draft short operational and stakeholder responses.
- `okan-codex-workflow`: follow the general engineering workflow.
- `okan-task-write-process`: update source-of-truth artifacts.
- Skills do not permit staging, committing, pushing, posting comments, or
  sending messages unless explicitly requested.

## Style

- Return only the requested artifact unless explanation is requested.
- State the concrete change, blocker, risk, or next action.
- Keep wording concise, direct, and easy to forward.
- Never claim validation without inspected command output.
- Do not mention AI assistance.
- End with one verification request when the recipient must check something.
