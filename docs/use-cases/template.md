# Use Case Title

## 1. Summary

Describe the scenario in a few sentences. Keep this focused on what the actor
is trying to accomplish and why it matters.

## 2. Actor

Name who or what initiates the use case, such as a user, MCP host, MCP client,
or Local Orchestration Router (LOR) itself.

## 3. Scenario

Describe the concrete situation that triggers this use case. Include relevant
context, but avoid implementation details.

## 4. Flow

1. List the main path in order.
2. Keep each step observable from the actor or system perspective.
3. Stop when the expected outcome is reached.

## 5. Expected Outcome

Describe what should be true when the flow succeeds.

## 6. Related Feature Specs

- Link to relevant feature specs under `docs/feature-specs/`.

## 7. Open Questions

- Track unresolved scenario details that need product or implementation
  decisions.
