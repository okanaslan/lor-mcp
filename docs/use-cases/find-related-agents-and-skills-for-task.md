# Find Related Agents And Skills For A Task

## 1. Summary

A Codex user asks a Codex agent to complete a task in a workspace where Agentic
Router MCP is installed and configured. The task explicitly instructs the agent
to use LOR MCP to find related agents or skills, inspect their
details, and use them when relevant.

## 2. Actor

Codex user.

## 3. Scenario

The Codex user opens Codex in a workspace where LOR MCP is available.
The user gives an arbitrary Codex agent a task and includes instructions to find
related agents or skills for that task through LOR MCP. The current
agent uses the MCP tools to discover relevant catalog entries, inspect their
details, and decide whether to use a skill, contact another agent, or continue
without additional help.

## 4. Flow

1. The Codex user opens a workspace where LOR MCP is installed and
   configured.
2. The Codex user gives the current Codex agent a task.
3. The task description instructs the current agent to find related agents or
   skills for the task using LOR MCP.
4. The current agent calls LOR MCP to find matching catalog entries.
5. Local Orchestration Router (LOR) returns relevant agent or skill candidates, a no-match result,
   or a conflict result.
6. The current agent fetches detailed metadata for promising candidates.
7. The current agent reviews returned metadata, recommendation explanation, and
   any prompt or contact guidance.
8. The current agent decides whether to use a skill directly, contact another
   agent, or continue alone.
9. If another agent is relevant, the current agent prepares a task-specific
   prompt for that agent.
10. The current agent continues the original task using the selected support, or
    continues without additional agents or skills when no useful entry exists.

## 5. Expected Outcome

The current Codex agent can identify relevant catalog entries and understand
enough detail to use a skill or coordinate with another agent. If no useful
entry exists, the current agent continues without inventing unavailable agents
or skills.
