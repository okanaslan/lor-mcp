# Feature Title

## 1. Summary

Describe the feature in a few sentences. Keep this limited to the core purpose
and expected outcome. Include the current lifecycle status, such as draft,
planned, in progress, blocked, or completed.

## 2. Goals

- List the concrete outcomes this feature must achieve.
- Keep each goal specific enough to verify later.

## 3. Non-Goals

- List related work that is intentionally out of scope.
- Use this section to prevent the feature from expanding beyond its purpose.

## 4. Functional Requirements

Document the required behavior of the feature. Use clear, testable statements
that describe what the system must do.

## 5. User Stories / Use Cases

Optional. Add when user-facing workflows or agent workflows need to be made
explicit.

## 6. Data Model

Optional. Add when the feature introduces or changes persisted data, schemas,
entities, or important in-memory structures.

## 7. Error Handling

Optional. Add when the feature has meaningful failure modes, fallback behavior,
validation errors, or recovery rules.

## 8. Security and Permissions

Optional. Add when the feature affects access control, credentials, tool
permissions, user data, filesystem access, or external services.

## 9. Open Questions

Optional. Track unresolved questions that must be answered before or during
implementation.

## 10. Decision Log

Record important decisions made while shaping the feature. Include the date,
decision, and short reason when useful.
