# Tech Spec Title

## 1. Summary

Describe the technical topic, implementation concern, or decision being
discussed. Keep this focused on the core technical issue and expected outcome.

## 2. Context

Explain the current system state, related feature specs, constraints, and
assumptions that shape the technical discussion.

## 3. Goals

- List the concrete technical outcomes this spec should achieve.
- Keep each goal specific enough to validate during implementation.

## 4. Non-Goals

- List related technical work that is intentionally out of scope.
- Use this section to keep the discussion from expanding beyond the issue.

## 5. Proposed Design

Describe the recommended technical approach. Include architecture, data flow,
tool boundaries, storage behavior, interfaces, or algorithms when relevant.

## 6. Alternatives Considered

Document meaningful options that were considered and why they were not chosen.

## 7. Implementation Notes

Capture practical details that will matter during coding, such as file layout,
library choices, migration steps, local configuration, and sequencing.

## 8. Risks and Tradeoffs

List technical risks, operational risks, limitations, and tradeoffs introduced
by the proposed design.

## 9. Verification Plan

Describe how the technical design should be tested or validated during
implementation.

## 10. Open Questions

- Track unresolved technical questions that must be answered before or during
  implementation.

## 11. Decision Log

Record important technical decisions made while shaping the spec. Include the
date, decision, and short reason when useful.
